1. 模块概述

界面描述：大标题「风险报告」；副标题展示当前项目名；三个 Tab：报告中心、模板管理、定时任务；下方 router-view；布局挂载模板 Dialog、任务 Dialog、推送 Dialog。

一句话：项目上下文维护模板，在报告中心快速生成/导出，或通过定时任务经通知渠道推送；历史与执行日志可追溯。

1.1 状态流转（Mermaid）

模板 status：enabled ↔ disabled；任务 enabled：Switch 即 persist()；执行日志 success|partial|fail。

stateDiagram-v2
  [*] --> enabled: 模板/任务创建
  enabled --> disabled: 停用
  disabled --> enabled: 启用
  state 执行一次手动或定时 {
    [*] --> Running: execute
    Running --> success: 全渠道 OK
    Running --> partial: 部分渠道失败
    Running --> fail: 无成功/校验失败
  }

若下图未渲染，请检查网络（Mermaid 自 CDN 加载）或直接阅读 Markdown 源文件。

1.2 业务规则（模块级）

规则 M-1：切换项目时 reloadReportState() 从 yuyi-risk-report-proj-{projectId} 重载，数据不串项。
规则 M-2：系统模板 tenant_id===0 且 isSystem===true：不可删除、不可直接保存编辑。
规则 M-3：快速生成候选模板 = !deleted && status==='enabled' && periodType===quick.period。

1.3 交互流程

  1. 菜单进入风险管理 › 风险报告 → 默认 /risk/report/center。
  2. 点模板管理 → /risk/report/templates（SPA）。
  3. 点定时任务 → /risk/report/schedules。
  4. 切换顶栏项目 → 报告数据整体重载。

2. 报告模板管理

2.1 模板列表页

界面描述：说明文案；「新增模板」；表格：名称、类型、来源 Tag、状态 Tag、操作（系统：复制；项目：编辑/删除）。当前无：顶栏搜索、导入导出、分页、行内启停。

字段定义表（列表行）

字段名类型长度必填默认值校验规则
idstring64是rtpl-*/sys-*唯一
namestring64是—trim；上线：项目或同类型唯一
periodTypeenum—是—day/week/month/quarter/year
statusenum—是enabledenabled|disabled
isSystemboolean—是—系统模板 true
tenant_idany—是0/projectId—
deletedboolean—是false软删
chartTheme/chartSize/showDataLabel/sections详见 §2.2

交互流程

  1. 进入：visibleTemplates = allTemplates.filter(!deleted)。
  2. 新增：openTemplateCreate()。
  3. 编辑：非系统 → openTemplateEdit。
  4. 复制系统：copySystemTemplate → 名称加「（项目副本）」。
  5. 删除项目：Confirm → deleted=true；上线需 DELETE /api/report/templates/{id} 409 校验任务引用。
规则 T-L-1：isSystem||tenant_id===0 不可删，仅 Alert。
规则 T-L-2：软删，列表与快速生成过滤 deleted。
规则 T-L-3（上线）：删前校验 schedule_task.template_id。

8. DDL 示例（与 Markdown 全文一致）

CREATE TABLE report_template (
  id VARCHAR(64) PRIMARY KEY,
  name VARCHAR(128) NOT NULL,
  type VARCHAR(16) NOT NULL COMMENT 'day/week/month',
  config JSON NOT NULL,
  status VARCHAR(16) NOT NULL DEFAULT 'enabled',
  is_system TINYINT(1) NOT NULL DEFAULT 0,
  project_id BIGINT NULL,
  create_time DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
  update_time DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)
);

CREATE TABLE report_history (
  id VARCHAR(64) PRIMARY KEY,
  template_id VARCHAR(64) NOT NULL,
  file_path VARCHAR(512) NOT NULL,
  file_size BIGINT,
  generate_time DATETIME(3) NOT NULL,
  date_range JSON NOT NULL,
  project_id BIGINT NOT NULL
);

CREATE TABLE schedule_task (
  id VARCHAR(64) PRIMARY KEY,
  name VARCHAR(160) NOT NULL,
  template_id VARCHAR(64) NOT NULL,
  cron VARCHAR(64) NOT NULL,
  push_channels JSON NOT NULL,
  email_receivers VARCHAR(1024),
  dingtalk_mentions VARCHAR(512),
  status VARCHAR(16) NOT NULL DEFAULT 'enabled',
  project_id BIGINT NOT NULL
);

CREATE TABLE schedule_execution_log (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  task_id VARCHAR(64) NOT NULL,
  execute_time DATETIME(3) NOT NULL,
  status VARCHAR(16) NOT NULL,
  result_message TEXT,
  file_size BIGINT,
  push_result JSON
);

CREATE TABLE notification_channel (
  id VARCHAR(64) PRIMARY KEY,
  type VARCHAR(32) NOT NULL COMMENT 'email, dingtalk',
  name VARCHAR(128) NOT NULL,
  config JSON NOT NULL,
  status VARCHAR(16) NOT NULL DEFAULT 'enabled'
);
