1 · 引言与端到端业务流程

1.1 产品目标

为机关/园区物业管理场景提供统一门户：多项目租户隔离、风险指标采集与规则告警、实时事件与安全运营、消息触达与工作台工单、风险报告导出。本文档拆解各功能的界面、字段、交互、流程与验收，便于开发与测试直接落地。

1.2 主线：指标 → 规则 → 事件 → 消息 → 工单（闭环）

在项目上下文中：
   ① 创建/启用「风险指标」并配置数据源 ──► ② 可选「数据源测试」通过
                      │
                      ▼
   ③ 新建「风险规则」（绑定 metricCode + 阈值表达式），runMode=trial/production
                      │
        ├─ 试运行 ──► ④「试运行日志」记录命中/未命中摘要
        └─ 正式运行 ─.rules 触发后生成「实时事件」source=rule
                      │
                      ▼
   ⑤ 推送策略（事件推送配置）决定通知渠道与工单是否生成 / 工单类型
                      │
                      ▼
   ⑥「消息中心」产生未读，用户跳转「实时事件抽屉」查看规则快照与阈值
                      │
                      ▼
   ⑦ 关联工单进入「工作台」各视图跟进，直至办结（closed）
数据依赖：平台标准指标类型（SM-*）仅存于系统模板语义层；映射到本项目指标编码后才参与真实取值与规则计算。

应用框架

需求说明

提供统一壳层：左侧可折叠导航（多级子菜单）、顶部状态栏（北京时间、全局帮助、用户信息）、当前项目选择器（全局过滤数据口径）、面包屑由各页自行决定是否展示。响应式：窄屏下侧栏收起、表格横向滚动或使用弹性布局保底。

界面布局（文字）

  • 左侧：Logo + 收起按钮；纵向滚动的 el-menu（一级菜单可展开二/三级）。底部显示版本号（如 v2.0.0）。
  • 顶栏（主内容顶部固定条）：左侧留空或由子页标题填充；右侧依次「系统时间」「项目下拉」「消息铃铛+未读角标」「帮助」「头像+下拉（个人中心/系统设置/退出）」。
  • 主内容区：白色底，内边距统一；子路由 router-view 渲染。

字段定义 — 项目切换（持久化）

字段名类型必填说明示例
currentProjectId string 是 当前选中项目，切换后广播 yuyi-project-changed，各模块按项目拉数 p-huangpu
projectOptions[] array 是 用户有权限的项目列表（后续走接口） [{ id, name }]

交互规则

  • 折叠侧栏：图标按钮切换 collapsed；菜单仅保留图标或 tooltip 全称。
  • 项目切换：变更写入 localStorage，页面内列表/看板应立即刷新或通过事件总线失效缓存。
  • 校验：无（选择器仅展示合法枚举）。
  • 反馈：无全局 toast（避免打扰）；静默更新数据。

验收标准

  • ✓ 所有一级路由可由侧栏在 3 次点击内可达（当前「建设中」占位仍保留入口）。
  • ✓ 刷新页面后同一项目 persisted。
  • ✓ 视口宽度 < 768px 时侧栏仍可操作（折叠或可抽屉化，不出现样式塌缩）。

风险管理 · 风险指标 CRUD

需求说明

管理本项目内可观测度量：metricCode 由板块 + 分类 + 三位流水号自动生成；配置数据源类型（API/JDBC/MQ/Kafka 等占位）、采样频率/Cron；支持数据源连通性测试指标拉取试运行；停用指标需提示是否已被规则引用。

界面布局

  • 列表页：表格列含编码、名称、板块、数据源类型、单位、启用状态、被规则引用次数、操作（编辑 / 停用 / 测试）。
  • 表单抽屉：分左右或上下滚动区——基础信息、数据源配置分段（切换 type 动态展示 API / DB / MQ 字段）。底部「保存」「取消」「测试连接」。
  • 测试弹窗：展示最近一次请求耗时、返回值摘要、断言结果（前端 demo）。

交互规则

  • 保存前校验：名称非空；数据源字段按类型联动必填。
  • 「测试连接」：不改变指标正式状态；结果以对话框或结果区 toast。
  • 试运行日志页通过时间范围、metricCode、规则 id 检索（参见试运行日志小节）。

验收标准

  • ✓ 在同一项目内 code 唯一。
  • ✓ 停用指标后，已绑定的规则编辑页应展示警告或禁止「仅正式运行」。

风险管理 · 风险规则 CRUD

需求说明

将指标与布尔条件表达式绑定，支持主条件 + 附加条件（AND/OR）、时间窗、静默期、预警等级、事件分类、是否自动建单及工单类型；运行模式试运行/正式；支持规则测试（mock 结果）、版本历史（至少展示版本号、变更人、时间）、另存为项目模板。

界面布局

  • 列表：规则名、指标、等级标签、运行模式、最近触发、操作（编辑/复制/停用/查看版本）。

风险管理 · 规则模板库

需求说明

本项目模板：用户将规则「另存为模板」，仅当前 projectId 可见，列表展示模板名、适用板块、引用指标（含项目编码）。系统模板：tenant_id=0，引用平台标准指标类型（SM-*），不直接带项目指标编码；用户「使用」时在映射弹窗选择本项目指标；支持「另存为项目模板」生成副本。

界面布局

  • 风险规则页双区域：上「本项目模板」卡片栅格 + 操作；下「系统模板」卡片 + 「管理」入口。

标准指标类型字典：平台级枚举（key、名称、默认单位、预期数据源类型）；由运营/超管维护（当前前端为静态资源 + 后续可接管理 API）。系统模板 CRUD：仅角色 super_admin（或等价）可见「新增/编辑/删除」；新建表单内标准类型下拉仅展示名称，附文案说明平台预置与项目无关。

验收标准

  • ✓ 非超管无法访问系统模板写接口与 UI 按钮（前后端双重校验）。
  • ✓ 修改 SM 字典需评估旧系统模板的兼容与迁移策略。

风险报告

需求说明

支持按项目输出风险运营类报告：模板定义章节结构、占位符与导出格式；报告生成为一次性任务，产生文件与状态；定时任务按 Cron 周期自动触达并可选邮件推送链接。所有数据按当前 projectId 过滤。

界面布局

  • 模板管理：表格列（模板名、类型、更新人、更新时间、操作）；抽屉表单含「基本信息」「变量声明」「导出设置」。
  • 报告中心：筛选（模板/时间/状态）+ 表格（批次号、模板、生成时间、状态、操作下载/重试）。
  • 定时任务：表格 + 弹窗配置 Cron 表达式、下一次执行预览、失败告警邮箱。

字段 — 报告模板（节选）

字段名类型必填说明示例
namestring是模板名称能耗风险月报
reportTypeenum是统计/明细/混合统计
outputFormatenum是PDF/XLSX/DOCXPDF

字段 — 定时任务（节选）

字段名类型必填说明示例

cronExprstring是调度表达式0 8 1 * *

enabledbool是启停true

业务流程

选择模板 → 预览变量 → 触发「立即生成」→ 异步任务写入报告中心 → 成功可下载 / 失败可重试
定时：Cron 触发 → 同生成链路 → 可选邮件带签名下载链接（短期有效）

交互规则

  • 长时间生成：按钮 loading + 轮询或 WebSocket 进度（P1）。
  • Cron 合法性：保存前服务端解析，前端展示「下次 5 次执行时间」预览。

验收标准

  • ✓ 模板删除前检查是否被定时任务引用。
  • ✓ 导出文件带水印或机构标识（按安全要求）。
  • ✓ 报告仅含当前项目数据，无交叉泄漏。

事件工作台 · 工单

需求说明

同源组件多路由：报修工单（全量）、我的待办、我发起的、我的已办、办结事宜。支持筛选、排序、单条「接单/转派/完成」类弹窗、批量选择 + 批量转派/关闭（权限控制）；行内打开工单详情抽屉（时间线、评论、关联事件跳转）。

字段（工单 — 节选）

字段名类型必填说明示例
idstring是工单号WO-20260427001
categoryenum是risk/repair/inspectionrisk
statusenum是pending/processing/done/closedpending
assigneeIdstring|null否责任人u-zhang
eventIdstring|null否风险类关联事件evt-xxx

业务流程（简）

待办打开工单 → 接单 → 处理中上传凭证/备注 → 完成 →（可选复核）→ 办结归档
批量：勾选多条 → 批量转派（选人弹窗）→ 确认 → 成功/失败分项 toast

交互规则

  • 单条操作：根据状态机灰显非法操作（如已完成不可再「接单」）。
  • 批量：至少选 1 条；部分失败时列表刷新并展示失败原因列。
  • 详情抽屉：Tab「概况 / 时间轴 / 关联事件」；关联事件点击跳转事件详情。

验收标准

  • ✓ 待办视图仅包含「本人为当前责任人」且未完成单据。
  • ✓ 批量操作需二次确认并展示影响条数。
  • ✓ 「我发起的」含用户手工发起及规则自动生成但需跟踪的工单（与业务定义一致）。
