2 · 产品概述与全局口径

2.1 背景与目标

  • 背景：园区/机关安防与风险运营需统一配置台、运营台、工作台。
  • 目标：可配置监测与处置；报表生成与多渠道推送；当前项目数据隔离。
  • 现状：核心前端可端到端演示；后端与 RBAC 需按本文补齐。

2.2 整体业务流程（文字）

  1. 配置：风险指标 → 风险规则（试运行）→ 事件分类（超管）。
  2. 接入：实时事件。
  3. 推送：事件推送配置 × 通知方式配置。
  4. 处置：工单工作台多视图。
  5. 分析：风险看板；风险报告（生成/定时/历史）。

2.3 权限矩阵（当前代码）

能力演示用户项管建议超级管理员
菜单「事件分类」隐藏隐藏可见 /risk/event-categories
工作台导出isAdmin=false disabled允许允许
其余页未接登录默认可操作；上线须路由守卫 + JWT
useSystemAdmin.js · isSystemSuperAdmin 默认 true；requiresSystemAdmin 未在 beforeEach 强校验——上线必须补。

2.4 全局交互与异常（详尽）

类别约定
网络失败统一：操作失败，请稍后重试
表单校验字段下方红线（Element Plus rules）
业务失败ElMessage.error(message)；弹窗默认不关
引用/并发冲突列出冲突对象；禁止静默；可选 409 / 423
分页默认 10 · pageNum/pageSize · sizes 10/20/50
三组角色（产品）超管 tenant_id=0 全局字典/模板；项管 当前项目配置；业务用户 本人相关数据域

1 · 模块概述

定位：风险报告中心为管理层与运营提供周期性风险数据分析报告；支持自定义模板、定时生成并推送钉钉/邮件，以及手动快速生成与历史归档。

  • 核心价值 A：按日/周/月自动输出，减轻人工统计。
  • 核心价值 B：PDF / Excel / Word 导出，便于汇报归档。
  • 核心价值 C：与通知方式配置的渠道账户打通（Webhook/SMTP 等）。

3.1.3 交互流程
  1. 进入页面 — 动作：打开模板 Tab。系统：GET /api/report/templates?page=1&pageSize=10 + 筛选 query。反馈：表格渲染 Skeleton → 数据。权限：业务用户隐藏新增/导入/导出及删除按钮。
  2. 筛选 — 动作：填条件点【搜索】。系统：页码复位 1，带参请求。反馈：刷新表格。【重置】清空条件再请求。
  3. 启用/停用 — 动作：点启用/停用。系统：ElMessageBox.confirm 文案：确认[启用/停用]模板「{name}」？停用后将不能用于生成报告。 → PUT /api/report/templates/{id} 更新 status。反馈：成功 Toast「操作成功」并刷新；失败 Toast 后端 message。
  4. 编辑 — 打开弹窗（§3.2），GET /api/report/templates/{id} 拉详情，保存后刷新列表。
  5. 复制 — 弹窗「复制」模式：预填原配置；名称后缀 _副本，冲突则追加数字。POST /api/report/templates。
  6. 删除 — Confirm：确定删除「{name}」？删除后不可恢复。 若任一启用中的定时任务引用 → 禁止删除，Toast/Modal：无法删除，该模板已被 X 个定时任务使用… → 409。否则 DELETE /api/report/templates/{id}。
3.1.4 业务规则
规则 1：系统预置三个模板（标准日/周/月），不可物理删除；界面可停用。
规则 2：同一tenant_id/project_id下，同类型模板名称唯一。
规则 3：停用模板不出现在快速生成下拉，也不可被新建/启用中的定时任务选用（已绑定任务的展示策略由产品核定：禁用编辑或告警）。
规则 4：与 3.1.3 删除拦截二选一已定稿：（A）被引用则 409 禁止删；或（B）删除时级联将任务 template_id=NULL 并停用任务。本节默认采纳 A（与 TC-TL-03 一致）；若选 B，须同步修订验收与 UI 文案。
3.1.5 异常处理
场景前端后端

删除冲突Modal 提示引用数409 Conflict + message

4 · 报告生成与预览（快速生成）

Path：/risk/report/center（快速生成区在主内容区）；若独立页则对应 PRD 「/report/quick」等价能力。

4.1.1 界面布局
  • 报告周期：日/周/月 + 自定义日期范围。
  • 模板下拉：仅「当前项目 + 启用 + 周期类型匹配」模板。
  • 【生成报告】；中区 HTML 预览；底部【导出 PDF/Excel/Word】；无内容时空态：暂无报告，请点击生成。
4.1.2 交互流程 — 生成
  1. 动作：选周期 + 模板 →【生成】。系统：POST /api/report/generate：
{
  "templateId": "tpl_001",
  "dateRange": { "start": "2026-05-01", "end": "2026-05-07" },
  "projectId": "<current>"
}
  1. 响应：htmlContent 或预览 URL→前端挂载预览；Loading「报告生成中…」禁用按钮。
  2. 失败：Toast「报告生成失败，请稍后重试」+ 可追溯 requestId。
导出
  • 点击 PDF/Excel/Word→文件流下载；文件名 {模板名}_{区间}_{timestamp}.{ext}。
  • 若预览为空点击导出：先静默生成再继续导出（产品与性能须评估缓存）。
4.1.3 业务规则
规则 8：日报 + 自定义时，跨度默认单日；周报建议整自然周（可配置是否强制校准）。
规则 9：报告数据不包含试运行事件（与正式工单/事件聚合）。
规则 10：临时预览/缓存 TTL 1 小时；归档历史永久策略见 §5（90 天文件保留为存储侧策略）。
4.1.4 异常处理
  • 所选区间无事件→「所选时间范围内无任何事件，无法生成报告」。
  • 模板 config 损坏→「模板配置异常，请联系管理员」。

6.1.3 立即执行
  1. Confirm：「确认立即执行？将生成报告并推送给接收人」。
  2. POST /api/report/schedules/{id}/execute；若执行中：423/409「任务正在执行」。
  3. 异步：Toast「任务已开始执行…」。
规则 13：同项目任务名唯一。

规则 16：失败自动重试 ≤2 次，间隔 5 分钟（服务端策略；与推送「不重试钉钉配置错」可按渠道分支）。

7 · 推送方式整合

  • 7.1 数据来源：渠道由 通知方式配置 统一管理；表单多选调用 GET /api/notification/channels?status=enabled。

CREATE TABLE report_template (
  id VARCHAR(32) PRIMARY KEY,
  tenant_id VARCHAR(32) NOT NULL,
  name VARCHAR(64) NOT NULL,
  type VARCHAR(10) NOT NULL,
  config JSON NOT NULL,
  status TINYINT DEFAULT 1,
  create_time DATETIME,
  update_time DATETIME

CREATE TABLE report_history (
  id VARCHAR(32) PRIMARY KEY,
  tenant_id VARCHAR(32) NOT NULL,
  template_id VARCHAR(32),
  file_path VARCHAR(255),
  file_size INT,
  generate_time DATETIME,
  date_range_start DATE,
  date_range_end DATE
);
CREATE TABLE schedule_task (
  id VARCHAR(32) PRIMARY KEY,
  tenant_id VARCHAR(32) NOT NULL,
  name VARCHAR(64) NOT NULL,
  template_id VARCHAR(32) NOT NULL,
  cron VARCHAR(64) NOT NULL,
  push_channels JSON,
  email_receivers TEXT,
  dingtalk_mentions VARCHAR(255),
  status TINYINT DEFAULT 1,
  create_time DATETIME,
  update_time DATETIME
);
CREATE TABLE schedule_execution_log (
  id VARCHAR(32) PRIMARY KEY,
  task_id VARCHAR(32) NOT NULL,
  execute_time DATETIME,
  status VARCHAR(20),
  result_message TEXT,
  file_size INT,
  push_result JSON,
  INDEX idx_task (task_id)
);

CREATE TABLE report_template (
  id VARCHAR(64) PRIMARY KEY,
  name VARCHAR(128) NOT NULL,

  status VARCHAR(16) NOT NULL DEFAULT 'enabled',
  is_system TINYINT(1) NOT NULL DEFAULT 0,
  project_id BIGINT NULL,
  create_time DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
  update_time DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)

  date_range JSON NOT NULL,

7 · 验收测试标准（节选）

场景：项目上下文隔离
  假设 用户在顶栏切换项目
  那么 报告任务与执行日志存储键切换且无交叉数据

场景：定时任务执行-邮件未配置
  假设 邮件 SMTP 或收件人为空
  当 立即执行
  那么 toast 与历史状态为 fail 或 partial 且文案准确

场景：非超管访问事件分类
  给定 isSystemSuperAdmin 为 false
  当 打开 /risk/event-categories
  那么 页面阻止维护并提示

场景：工作台导出权限
  给定 isAdmin 为 false
  那么 「导出当前列表」disabled

回归提示：改动 request.js、useCurrentProject、riskReportShared、scheduleExecutionMock、LeftMenu、规则保存冲突等，须跑 §3 冒烟与报告全流程。
