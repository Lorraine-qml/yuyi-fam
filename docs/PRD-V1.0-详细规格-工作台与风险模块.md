# 禹翼机管局平台 — 详细规格（工作台 · 风险 · 附属配置）

本文档是对 `PRD-禹翼机管局平台-V1.0-开发测试全量.md` 的**深化补充**。凡与实现对齐处有脚注说明；其余为**目标产品口径**（研发按此收口，必要时再回填主 PRD）。

## 〇、全局编写规范（适用所有页面）

### 〇.1 路由与菜单约定

| 维度 | 约定 |
|------|------|
| **路由路径** | 与 `src/router/index.js`、`LeftMenu.vue` 一致；本文在页面节首给出完整 path。 |
| **菜单层级** | 写明「一级模块 › 二级分组 › 三级菜单项」与本页 path。 |
| **视图切换** | 推荐：同源组件切换时仅变更 **query**（如 `?view=`）或 **path**（五条独立 path），**禁止整页浏览器刷新**；数据由前端重新请求表格接口。 |

### 〇.2 表单 / 弹窗字段表列说明

每张字段表统一包含：**字段名｜控件类型｜长度/范围｜必填｜默认值｜校验规则｜备注**。

### 〇.3 联动与反馈

| 场景 | 行为 |
|------|------|
| **提交成功** | `ElMessage.success` 简述结果；列表页 **重新拉取当前页**（或乐观更新后与服务器对齐）。 |
| **提交失败（业务码）** | `ElMessage.error` 展示后端 `message`；表单不关闭（除非明确设计为关闭）。 |
| **网络请求失败** | **统一全局文案**：「操作失败，请稍后重试」（可在监控中打点，不向普通用户暴露技术细节）。 |
| **校验不通过** | 字段 `prop` 对应项 **下方红线 + 文案**（Element Plus `rules`）；提交按钮可 `loading` 后恢复。 |

### 〇.4 列表 / 表格通用约定

| 项 | 约定 |
|----|------|
| **筛选类型** | 标明 **精确** / **模糊** / **多选** / **枚举单选** / **日期范围**。 |
| **分页** | 默认 **每页 10**；`pageNum`/`pageSize`（或后端约定同名参数）；支持 `sizes` **10 / 20 / 50**。 |
| **排序** | 列明 **默认可排序列** 与 **默认排序**（无则写「不提供排序」）。 |

### 〇.5 权限模型（三组角色）

> 后端以 **租户 + 角色** 为准；`tenant_id = 0` 表示平台超级租户（全局系统管理员）。以下为**可见/可操作范围**产品口径。

| 角色 | 数据范围（读） | 典型操作限制 |
|------|----------------|----------------|
| **超级管理员**（tenant_id = 0 或等价系统标识） | 全平台模板、全局事件分类字典、任意项目（切换项目后） | 管理系统规则模板；维护事件分类；跨项目审计；工作台「全量导出」类能力 |
| **项目管理员** | 当前 **project_id**（当前选中项目）下全量工单/事件/规则/指标… | 配置指标规则、导出、批量转派等高权限操作（无全局模板写入时受后端约束） |
| **业务用户** | **本人相关**：待办（责任人=己）、发起的、参与的工单链；事件按项目成员可见策略 | **不可**导出全量列表（若产品上需限制）；不可编辑系统模板/全局字典 |

前端演示：`useSystemAdmin` 中 `isSystemSuperAdmin`、工作台 `isAdmin` 等与上表对齐后方可上线。

### 〇.6 状态流转（摘录）

| 实体 | 状态集合 | 说明 |
|------|----------|------|
| **风险指标** | 启用 / 停用 | 停用后关联 **正式规则** 不得触发新业务事件（或后端自动 cascade 停用规则，需选一种并实现） |
| **风险规则** | 启用 / 停用 × 运行模式 **正式 / 试运行** | 试运行：不计入正式通知/工单口径（与后端约定）；同指标下同模式冲突见规则保存拦截 |
| **工单**（示例） | 待接单 → 处理中 → 已完成 / 已关闭 / 已取消（误报） | 具体枚举以后端工单表为准；列表筛选与视图定义依赖此状态集 |
| **定时任务 / 报告** | 见主 PRD §3.9 | — |

### 〇.7 错误与异常处理（汇总）

| 类型 | 处理 |
|------|------|
| 网络失败 | 见 §〇.3 |
| 校验失败 | 字段级飘红 |
| **并发 / 引用冲突**（如删除被引用的指标） | `ElMessageBox` 或 `ElNotification`：**展示冲突对象列表**（如规则名称、编码），禁止静默失败 |
| **乐观锁 / 版本**（可选） | 保存返回 409 时提示「数据已被他人修改」并刷新 |

### 〇.8 验收用例格式

每条使用 **Given / When / Then**（等价 Gherkin），每页面 **不少于 3 条**。

## 3.4 页面名称 · 安全管理 › 事件工作台（工单运营台）

**组件**：`WorkOrderWorkbenchPage.vue`  
**路由与 `meta.workOrderView`**：`path` 为 `/security/workbench/:view`，其中 `view` 取 `all` | `todo` | `initiated` | `done` | `closed`；非法值回退至 `all`。历史路径 `/security/workbench/repair` 重定向至 `/security/workbench/all`（保留 query/hash）。

| 菜单层级 | Path | view |
|-----------|------|------|
| 安全管理 › 事件工作台 › **报修工单** | `/security/workbench/all` | `all` |
| 安全管理 › 事件工作台 › **我的待办** | `/security/workbench/todo` | `todo` |
| 安全管理 › 事件工作台 › **我发起的** | `/security/workbench/initiated` | `initiated` |
| 安全管理 › 事件工作台 › **我的已办** | `/security/workbench/done` | `done` |
| 安全管理 › 事件工作台 › **办结事宜** | `/security/workbench/closed` | `closed` |

**脚注**：实现为 **单一路由** `workbench/:view` + `beforeEnter` 写入各视图的 `meta`；顶部 Tabs 绑定完整 path，切换视图为 SPA 内导航，不重载文档。

### 功能概述

同一运营页在五种视角下切换，对工单数据集应用 **视图默认筛选**，并与用户自选筛选、分页、批量操作组合。

### 主要用途

值班长/管理员掌握全量与超时；经办人查看待办与已办；发起人跟踪进度；归档查阅办结工单。

### 界面布局（文本 + ASCII）

- **区块 1 — 页头**：标题「事件工作台」；右侧「更多」（导出、重置筛选）+「自定义列」（部分视图）。
- **区块 2 — 主 Tab**：五视图切换（与菜单项一一对应）。
- **区块 3 — 说明条**：视图相关一句业务说明（可复制 `pageDesc`）。
- **区块 4 — 筛选卡片**：表单 inline，单行多列 + 查询/重置。
- **区块 5 — 批量操作条**：随 `viewMode` 变化（见下文业务规则）。
- **区块 6 — 表格卡片**：可多选列、分页、排序（若产品上启用）、操作列。
- **区块 7 — 弹窗/抽屉**：处理、批量处理、详情抽屉等挂载于本页或子组件。

```
+----------------------------------------------------------+
| 事件工作台                                   [更多v][列v]|
+----------------------------------------------------------+
| [报修工单][我的待办][我发起的][我的已办][办结事宜]  (Tabs)|
+----------------------------------------------------------+
| 说明文案一行                                              |
+----------------------------------------------------------+
| 筛选: 状态 紧急 类型 关键词 日期   [搜索] [重置]         |
| --------- 批量区（依视图） ----------                     |
| [表格 ==============================]                     |
| 分页                                                      |
+----------------------------------------------------------+
```

### 五视图：含义与数据筛选逻辑

| view | 用户含义 | 默认数据范围（服务端宜实现） |
|------|----------|------------------------------|
| `all` | 报修工单 / 全量视角 | **当前项目**下全部工单（仍尊重业务用户权限时收缩为「可 see」集合） |
| `todo` | 我的待办 | **当前责任人 = 当前用户** 且 状态 ∈ {待接单, 处理中}（枚举名以后端为准） |
| `initiated` | 我发起的 | **发起人 = 当前用户** |
| `done` | 我的已办 | 用户曾在流程中处理过且 **未归档至 closed**（或：最近处理人为本人的「已完成待办结」口径，以后端为准） |
| `closed` | 办结事宜 | 状态 ∈ {已完成, 已关闭, 已取消} 等终结态 |

**URL**：路径为 `/security/workbench/:view`；亦可通过 `router.push` 仅变更路径或 query（如 `focusWo`）；不重载页面。旧书签 `/security/workbench/repair` 自动跳转到 `/security/workbench/all`。

### 筛选区字段定义（目标为多选时请后端支持数组参数）

| 字段 | 控件 | 筛选类型 | 说明 |
|------|------|----------|------|
| 状态 | `el-select` **multiple** | 精确 · 多选 | 工单状态全集子集 |
| 紧急程度 | `el-select` **multiple** | 精确 · 多选 | 紧急 / 重要 / 一般（与 Tag 色系一致） |
| 工单类型 | `el-select` radio 或 single | 精确 · 单选 | 例：报修 / 风险处置… |
| 关键词 | `el-input` | 模糊 | 工单编号 **或** 摘要 **或** 位置 **或** 责任人 |
| 日期 | `daterange` | 范围 | 创建时间或更新时间（产品需固定一种并在接口文档写明） |

**按钮**：**搜索**（触发表格查询）、**重置**（清空条件并查询）。

**脚注**：当前前端部分筛选项为 **单选** dropdown，升级为多选时改 `filters` 结构与 API。

### 列表区列定义（产品口径）

| 列 | 说明 |
|----|------|
| 工单编号 | 支持复制 |
| 类型 | 枚举展示 |
| 紧急程度 | **徽章**：紧急=红 / 重要=橙 / 一般=灰 |
| 摘要 | 省略+tooltip |
| 位置 | — |
| 责任人 | — |
| 创建时间 | 格式化 `YYYY-MM-DD HH:mm` |
| 处理时限 | **剩余时长**倒计时或截止时刻；超时展示 **「已超时」** 强调样式 |
| 状态 | Tag |
| 操作 | **处理**、**详情** |

**可选列**：工单类型可见性开关、发起人（`initiated` 视图）、处理时限可见性——与「自定义列」一致。

### 列表：筛选 / 分页 / 排序 / 操作显示条件（产品）

| 能力 | 规格 |
|------|------|
| 筛选字段 | 上表 |

### 批量操作（`all` / `todo` 等）

| 能力 | 条件 |
|------|------|
| 表格多选 | `el-table-column type="selection"` |
| 批量简单处理 | 至少一行选中 |

| 批量催办 | 同上 |
| 批量导出 | **仅管理员**；选中或按筛选导出（产品与法务确认范围） |

#### 批量简单处理 · 弹窗字段

| 字段 | 类型 | 长度 | 必填 | 默认值 | 校验 |
|------|------|------|------|--------|------|
| 统一处理结果 | radio / select | — | 是 | — | 枚举：完成 / 转派 / 关闭 |
| 处理说明 | textarea | max **200** | 否 | 空 | — |
| 转派人 | 用户远程搜索下拉 | — | 条件必填 | — | 当结果=转派时必填 |

**联动**：选「转派」显示转派人；否则隐藏。

**反馈**：成功 → 关闭弹窗、`ElMessage.success`、**刷新列表当前页**；失败 → 见全局错误策略。

#### 批量转派 · 弹窗字段

| 字段 | 类型 | 必填 | 校验 |
|------|------|------|------|
| 目标责任人 | 下拉搜索 | 是 | 必须在组织中 |

仅对选中行中状态为「待接单/处理中」的生效；其余行服务端返回 **`partial_success`** 时前端列出未转派条目。

### 单条 · 处理弹窗

| 字段 | 说明 |
|------|------|
| 操作选项 | **完成并关闭** / **转派给他人** / **催办** / **误报/取消** |
| 处理说明 | 选填，max **200** |
| 转派人 | 「转派」时展示，远程搜索 |

**提交后**：更新状态、刷新列表、可选写操作记录时间线。

### 工单详情 · 抽屉

| 分区 | 内容 |
|------|------|
| 基本信息 | 编号、类型、等级、摘要、位置、责任人、时限、状态… |
| 处理流程 | 时间线组件 |
| 关联事件 | 风险类工单展示关联 **事件名称、规则编号/名称** |
| 操作记录 | 与推送、转派、催办相关日志 |

### 导出（仅管理员）

**Excel 列**：工单编号、类型、紧急程度、摘要、位置、责任人、创建时间、完成时限、状态、处理结果说明。

### 权限矩阵（本页）

| 功能 | 超级管理员 | 项目管理员 | 业务用户 |
|------|------------|------------|----------|
| 五视图菜单入口 | ✓（含全项目） | ✓ | ✓ |
| 「报修工单」见全量 | ✓ | ✓（本项） | 仅本人相关（策略可配） |
| 批量导出 / 导出选中 | ✓ | ✓ | ✗ |
| 批量误报关闭（若保留） | ✓ | ✓（可配） | ✗ |

### 业务规则与约束

1. **视图优先级**：视图默认 Scope **与** UI 筛选 **AND**。  
2. **批量转派** 过滤非法状态，避免静默成功。  
3. **超时展示**与 SLA 计算以后端时钟为准。

### 错误与异常处理

符合 **§〇.7**；转派并发：若责任人已变更，返回明确冲突文案。

### 验收测试用例

1. **Given** 用户为项目管理员且存在跨多人工单  
   **When** 进入「报修工单」视图  
   **Then** 可见当前项目下全部（或策略内全部）工单，且分页正确。

2. **Given** 用户为普通经办人  
   **When** 打开「我的待办」  
   **Then** 仅含责任人=本人且状态为待接单或处理中的记录。

3. **Given** 用户勾选多条「处理中」工单并批量转派给 B  
   **When** 提交成功  
   **Then** 责任人更新为 B，且 B 在「我的待办」中可见该批工单。

4. **Given** 一条工单已超过处理时限  
   **When** 列表渲染  
   **Then** 「处理时限」列显示「已超时」醒目样式。

5. **Given** 非管理员用户  
   **When** 查看批量区或更多菜单导出  
   **Then** 导出能力不可用（disabled 或菜单隐藏）。

## 3.5 页面名称 · 风险管理 › 风险看板

**路由**：`/risk/dashboard`  
**菜单**：风险管理 › **风险看板**

### 功能概述

以项目为边界展示风险 KPI、趋势与分布；支持下钻实时事件列表与处置入口。

### 主要用途

运营侧快速掌握态势、定位高发板块与等级、推动闭环。

### 界面布局（ASCII）

```

+----------------------------------------------------------------+
```

### 字段定义 · 顶部筛选（无表单提交，改动即触发查询或点刷新）

| 字段 | 类型 | 备注 |
|------|------|------|
| 时间 | `el-radio-button`：**今天 / 近7 / 近30 / 自定义** | 自定义为 `daterange` |

**脚注**：当前仓库 `RiskDashboard.vue` 中板块、等级或为 **单选**；本节为**升级目标**，与图表接口参数对齐后改造。

### KPI 卡片

| KPI | 下钻语义 |

| 处理中 | 对应状态 |

### 「最新风险事件」列表（5 条）

列：等级、标题、时间、位置、**处置** 快捷按钮（打开事件 Drawer 或工单创建）。

### 数据刷新策略

| 时机 | 行为 |
|------|------|
| 初次进入 | 并行请求 KPI + 各图 + 列表 |
| 时间/板块/等级变更 | 重新查询 |
| **自动刷新** | 可选，**默认 30s 轮询**；用户开关存 `localStorage` 或偏好接口 |

### 权限矩阵

| 能力 | 超级管理员 | 项目管理员 | 业务用户 |
|------|------------|------------|----------|

### 业务规则与约束

口径需与 **`/risk/trial-logs`**、 **`/risk/metrics`** 触发记录一致：试运行事件是否计入 KPI 须有统一开关策略。

### 错误与异常处理

### 验收测试用例
